## ⏰ Project Rules

### 1. 매일 Pm 05:00 일일 스크럼 회의 참석하기
- 현재 진행상황 및 차후 계획 보고
- 이슈 발생 상황 보고 

### 2 Git Commit Convention 지키기
- 기본 형식 : [commit type]: [commit message]
- commit type
  - feat : 기능 구현
  - fix : 버그 수정
  - docs : 문서 작업
  - refactor : 리팩토링
  - test : 테스트
  - chore : 기타 작업

### 3. Branch 생성 규칙
- 기본 형식 : [type]/[feature]
- 기능 구현이 완료되면 Main Branch PR 남기기
- PR 후 Merge된 브랜치는 삭제하기
